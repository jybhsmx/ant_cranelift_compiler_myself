## 行为准则
本项目及所有参与者均受行为准则约束。参与本项目即表示你同意遵守相关条款。

## 我该如何贡献？
提交信息必须按照以下规范执行  

<类型>(<作用域>): <补充>  
作用域可省略  
类型必须是下列之一:  

- feat：一个新功能
- fix：bug 修复
- docs：编辑文档
- style：不影响代码含义的更改 (空格、格式、缺少分号等，不是 css 的更改)
- refactor：既不修复 bug 也不添加特性的代码更改
- perf：提高性能的代码更改
- test：添加缺失的或纠正现有的测试

### 报告 Bug
- 在 [issues](https://github.com/ant-pl/ant_cranelift_compiler/issues) 中搜索是否已有相同的 Bug。    
- 如果没有，请提交一个新 issue，包含复现步骤、预期结果和实际结果。

### 提交代码 (Pull Request)
1. Fork 本仓库并从 `main` 分支创建你的功能分支。
2. 如果添加了新功能，请务必添加相应的测试用例。
3. 确保所有测试通过。
4. 提交 PR！
